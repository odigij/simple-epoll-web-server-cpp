Simple Epoll Web Server

 About

The Simple Epoll Web Server (SEWS) is a lightweight, single-threaded web server written in C++. It explores epoll-based, non-blocking I/O and event-driven architectures. SEWS is Linux-specific, relying on sys/epoll.h and POSIX system calls. It is worth to mention that this project is experimental.

 Technical Highlights

  • Epoll-based Event Handling – Uses epoll for efficient, non-blocking I/O.
  • Minimalist, Single-threaded Design – Event-driven programming without multi-threading.
  • Command-Line Configuration – Supports parameters:
    • --port (Set server’s listening port)
    • --max-request (Define max concurrent requests)
    • --epoll-count (Adjust epoll event count)
    • --timeout (Timeout of incomming connection)
    • --flags (Configuration flags)
    • --help (Show available options)
  • Static Content Serving – Handles HTML & CSS delivery.
  • OpenSSL Integration – Implements HTTPS support.

 Requirements

  • Linux OS
  • C++17 or later
  • GCC or Clang compiler
  • CMake
  • OpenSSL
  • P.S: If you have installed requirements on your platform with any package manager (apt, pacman, etc.) cmake going to find it via CMakeLists.txt.

󰆍 Building and Running

  • git clone <repository_url>
  • cd <repository_name>
  • mkdir build && cd build
  • cmake ..
  • make
  • openssl req -x509 -newkey rsa:4096 -keyout server.key -out server.crt -days 365 -nodes
  • ./sews --help to see options/args
  • ./sews ..args &

󱌣 Custom Routing

Normally SEWS comes up with default app folder with some handlers to display this page. To define custom routes, follow these steps:

  1. Create an entry.cpp file inside app directory.
  2. Include this necessary header: 
    #include "../include/app_entry.hpp"
  3. Implement the sews::initializeApp function: 
    void sews::initializeApp(sews::Router& router) {
            // Define your custom handlers & application logic here
          }
